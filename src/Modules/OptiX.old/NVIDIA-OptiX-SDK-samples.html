NVIDIA OptiX SDK Code Samples

The NVIDIA OptiX SDK provides a set of sample codes demonstrating a variety of use cases for the OptiX library. Also, provided is a utility library, sutil, which can be used for rapid development of simple OptiX applications. This code is released free of charge for use in derivative works, whether academic, commercial, or personal. (Full License)

An implementation of the OptiX library is required to run and compile code samples. Please obtain the NVIDIA OptiX library here.

The CMake cross-platform build system is used to configure and build the SDK. CMake 2.6.3 or higher must be installed. See platfrom specific build instructions for more details:

sample1 

This sample demonstrates a minimal OptiX API example. A solid green buffer is drawn. Basic features demonstrated include:
  • Usage of key RT objects; RTbuffer, RTcontext, and RTprogram.
  • Creation, setting and consumption of RTvariables.
  • Simple ray generation program.

collision 

OptiX is intended to be useful for non-rendering applications as well as graphics. This sample shows an OpenGL visualization from a collision detection and line-of-sight engine built on OptiX.

In this example, the engine simulates a number of moving objects, specifiable with –n; tracing rays against a static scene, specifiable with -o. The engine traces a number of collision probe rays, specified with –p, from each object center using a closest hit program, and line-of-sight rays between all pairs of objects using an any-hit program. The lines of sight are visiualized with GL lines and the collision rays are visualized in red or green.

hybridShadows 

Hybrid shadow algorithm demonstrated.
  • Renders world space positions in GL.
  • Passes buffer of first hit positions to OptiX where shadow rays are traced for each hit point.
  • Passes shadow mask back to GL for composite.

isgShadows 

Image Space Gathering Soft Shadows. isgShadows is an extension of the hybridShadows example that implements a smart image space blur to generate phenomenologically compelling soft shadows from the hard shadows generated by OptiX. See the paper for more details.
  • Renders world space positions in GL.
  • Passes buffer of first hit positions to OptiX where shadow rays are traced for each hit point.
  • Passes shadow mask back to GL for blurring and compositing.
  


Browse Files

isgReflections 

Image Space Gathering Reflections. isgReflections implements a smart image space blur to generate phenomenologically compelling glossy reflections from the perfectly specular reflection generated by OptiX. See the paper for more details.
  • Renders world space positions in GL.
  • Passes buffer of first hit positions to OptiX where reflection rays are traced for each hit point.
  • Passes reflection buffer back to GL for blurring and compositing.

swimmingShark 

This example shows BVH refitting. The marine animals are animated, deforming each frame. Since they have the same topology each frame, refitting instead of rebuilding is suitable. There is also a BVH over all the moving animals and over the scene as a whole. Specify the number of species with –f and the number of fish per species with –n.

zoneplate 

Illustration of how to do different types of anti-aliasing. Samples can be reconstructed into a final image using the following methods:
  • None
  • Sample Gather
  • Sample Scatter using floating point atomics - SM 2.0+ (e.g. Fermi class device) is required for this feature
  • Adaptive sampling based on a local contrast threshold. SM 2.0+ (e.g. Fermi class device) is required for this feature
It also demonstrates the use of a variety of reconstruction filters including box, triangle, windowed sinc, Mitchell, and gaussian filters.
